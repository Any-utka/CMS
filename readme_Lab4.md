## Цель работы

Освоить расширяемую модель данных WordPress: создать CPT (Custom Post Type), пользовательскую таксономию, метаданные с метабоксом в админ-панели, а также реализовать виджет для отображения данных на сайте.

### Ход выполнения

#### Шаг 1. Подготовка среды

1. В локальной установке WordPress переходим в папку ```wp-content/plugins```
2. Создаем директорию для нашего плагина ```PriorutyNotes```
3. Включаем отладку в ```wp-config.php```, установив ```define('WP_DEBUG', true);```

#### Шаг 2. Создание основного файла плагина

1. В папке плагина создаем файл ```priorityNotes.php```
![Img-1](https://imgur.com/EAkX786.png)
2. Добавляем в него метаданные плагина (название, описание, версию, автора)
3. Активируем плагин в админ-панели WordPress
4. Убеждаемся, что плагин активен и ошибок нет
![Img-2](https://imgur.com/30IcWs5.png)

#### Шаг 3. Регистрация Custom Post Type (CPT)

1. Добавляем функцию для регистрации CPT «Заметки» (Notes) с помощью ```register_post_type()```
2. Установливаем параметры CPT:
   - публичный (public),
   - поддержка заголовка, редактора, автора, миниатюры,
   - наличие архивной страницы,
   - иконка в админке
   - метки (labels) для удобства использования
3. Регистрируем CPT при инициализации WordPress с помощью хука ```init```

#### Шаг 4. Регистрация пользовательской таксономии

1. Добавляем функцию для регистрации таксономии ```«Приоритет» (Priority)``` с помощью ```register_taxonomy()```
2. Связываем таксономию с CPT «Заметки»
3. Установливаем параметры таксономии:
   - иерархическая (как категории),
   - публичная,
   - метки (labels) для удобства использования
4. Регистрируем таксономию при инициализации WordPress с помощью хука ```init```

   ```php
   function pn_register_notes_cpt() {
      ...
      $args = array(
        'labels' => $labels,
        'public' => true,
        'supports' => array('title','editor','author','thumbnail'),
        'has_archive' => true,
        'menu_icon' => 'dashicons-edit'
    );

    register_post_type('pn_note', $args);
   }

   add_action('init', 'pn_register_notes_cpt');
   ```

   ```php
      function pn_register_priority_taxonomy() {
         ...
      register_taxonomy('priority', 'pn_note', $args);
      }
      ```

#### Шаг 5. Добавление метабокса для даты напоминания

1. Создаем функцию для добавления метабокса в редактор CPT «Заметки» с помощью add_meta_box()

   ```php
   function pn_add_due_date_metabox() {
      ...
   }
   ```

2. В метабоксе добавляем поле для выбора даты напоминания (используйте HTML5 input type="date")

   ```php
   function pn_due_date_callback($post) {
      ...
      
    echo '<input type="date" name="pn_due_date" value="'.esc_attr($value).'" required />';
   }
   ```

3. Создаем функцию для сохранения значения даты при сохранении записи с помощью хука save_post.
4. Убеждаемся, что дата сохраняется корректно и отображается при редактировании записи.
5. Добавляем проверку ```nonce``` для безопасности при сохранении метаданных.
6. Сделаем поле даты обязательным для заполнения.
7. Добавляем валидацию даты (например, дата не может быть в прошлом). В случае ошибки сохранения, выведите сообщение об ошибке.

8. Отображаем дату напоминания в списке записей CPT «Заметки» в админке.

#### Шаг 6. Создание шорткода для отображения заметок

1. Создаем функцию для обработки шорткода
   - [priority_notes]
   - [priority_notes priority="Hight"]

2. В функции получаем и отображаем список заметок, соответствующих фильтрам
3. Регистрируем шорткод с помощью ```add_shortcode()```
4. Добавляем стили для оформления списка заметок
5. Обрабатываем случаи, когда нет заметок, соответствующих фильтрам, и выводим соответствующее сообщение: "Нет заметок с заданными параметрами"
   ![Img-2](https://imgur.com/FQdGCMx.png)
6. Если ```priority``` или ```before_date``` не указаны, шорткод должен отображать все заметки.
   ![Img-3](https://imgur.com/jWwILhG.png)
7. Добавляем форму для добавления заметок

#### Шаг 7. Тестирование плагина

1. Добавляем несколько заметок с разными приоритетами и датами напоминания
2. Присваиваем каждой приоритет ```(High/Medium/Low)``` и заполняем ```Due Date```

### Вывод

Плагин Priority Notes для WordPress позволяет добавлять и отображать заметки с приоритетом и датой напоминания. Реализованы CPT «Заметки», таксономия «Приоритет», форма добавления заметок на фронтенде с валидацией даты и стилизованный вывод заметок с фильтрацией по приоритету и дате. Плагин полностью интегрирован в тему, адаптивен и готов к использованию.

### Ответы на контрольные вопросы

1. Чем пользовательская таксономия принципиально отличается от метаполя? Приведи пример, когда выбрать таксономию, а когда - метаданные
   - Таксономия — это способ классификации записей (категории, теги, приоритеты), который позволяет группировать записи, фильтровать и создавать архивы. Таксономии имеют встроенные интерфейсы для выбора и отображения на сайте.
   - Метаполе (Custom Field) — это произвольное поле, привязанное к конкретной записи, хранящее дополнительную информацию, которая не обязательно используется для фильтрации или архива.

   Примеры:

   - Выбираем таксономию, если нужно фильтровать и выводить группы записей: например, приоритет заметки (High/Medium/Low) или жанр книги.
   - Выбираем метаполе, если нужна дополнительная информация для каждой записи, которая редко используется для фильтрации: например, дата напоминания, ISBN книги, рейтинг.

2. Зачем нужен nonce при сохранении метаполей и что произойдёт, если его не проверять?
   - Nonce — это специальный «токен безопасности», который проверяет, что запрос на сохранение данных пришёл от авторизованного пользователя с сайта, а не от злоумышленника.
   - Без проверки nonce: любой пользователь или внешний скрипт может отправить POST-запрос и изменить данные на сайте, что приведёт к уязвимости безопасности (CSRF).
3. Какие аргументы ```register_post_type()``` и ```register_taxonomy()``` чаще всего важны для фронтенда и UX (назови минимум три и объясни почему).
   - Для CPT (register_post_type()):
      - public — делает тип записи доступным на фронтенде; без него записи не будут отображаться в запросах.
      - has_archive — включает архивную страницу для CPT, что удобно для просмотра всех записей типа.
      - supports — определяет, какие элементы можно редактировать (заголовок, редактор, миниатюра), что важно для удобства работы с записью.

   - Для таксономии (register_taxonomy()):
      - hierarchical — определяет, как таксономия ведёт себя (категории или теги), важно для UX при выборе и фильтрации.
      - public — отображает таксономию на фронтенде, чтобы можно было фильтровать записи.
      - labels — настраиваемые метки интерфейса в админке, чтобы пользователю было понятно, что он выбирает.
