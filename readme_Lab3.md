## Цель работы

Научиться создавать собственную тему WordPress, разобраться в её минимальной структуре и принципах работы шаблонов.

### Ход выполнения

#### Шаг 1. Подготовка среды

1. В локальной установке WordPress переходим в папку ```wp-content/themes```
2. Создаем директорию для темы ```BookPunk```
3. Включаем отладку в ```wp-config.php```, установив ```define('WP_DEBUG', true)```

#### Шаг 2. Создание структуры

1. Обязательные файлы темы:

    - В папке темы создаем файл ```style.css``` с метаданными темы
    - После метаданных можно добавляем базовые CSS-правила
    - Создаем файл ```index.php``` - главный шаблон темы
2. Общие части шаблонов

- Корневая папка темы bookpunk/:

  - ```style.css``` – главный файл темы, описывает базовые стили сайта, содержит мета-информацию о теме (название, автор, описание).

  - ```functions.php``` – подключает все скрипты и стили, подключает файл theme-setup.php, добавляет дополнительные функции темы.

  - ```index.php``` – главный fallback-шаблон для вывода постов, если нет более специфического шаблона.

  - ```header.php``` – шапка сайта: логотип, меню навигации, подключение wp_head().

  - ```footer.php``` – футер сайта: копирайт, подключение wp_footer().

  - ```front-page.php``` – шаблон главной страницы: баннер, интерактивный фон, последние книги.

  - ```page.php``` – шаблон для обычных страниц (например, «О нас», «Контакты»), выводит контент через the_content().

  - ```single.php``` – шаблон одной записи (одной книги или статьи), выводит заголовок, автора, миниатюру и контент.

  - ```archive.php``` – шаблон для архивов постов: категории, список последних записей, вывод карточек книг.

- Папка template-parts/:

  - ```content-book.php``` – карточка книги/статьи: миниатюра, название, автор, краткое описание. Используется на главной странице, в архивах и на index.php.

- Папка assets/:

  - ```assets/css/style.css``` – основной CSS темы, стили для шапки, футера, баннера, карточек книг, формы контактов, магические эффекты.

  - ```assets/js/main.js``` – интерактивные эффекты: параллакс баннера, движение текста, магическая пыль, анимация кнопок и формы.

  - ```assets/images/library-banner.jpg``` – изображение баннера для главной страницы.

### Ответы на контрольные вопросы

1. Какие два файла являются обязательными для любой темы WordPress?

- style.css – главный файл стилей темы.
  - Содержит информацию о теме в шапке файла (комментарий с названием, автором, версией).

  - Без него WordPress не распознает тему.

- index.php – основной шаблон.

  - Отвечает за вывод страниц, если не найдены более специфичные шаблоны.

  - Без него тема работать не будет, даже если есть другие файлы шаблонов.

1. Как подключаются общие части шаблонов (header, footer, sidebar)?

WordPress использует специальные функции для подключения общих частей:

    - get_header(); – подключает header.php.

    - get_footer(); – подключает footer.php.

    - get_sidebar(); – подключает sidebar.php.
3. Чем отличаются index.php, single.php и page.php?

| Файл | Назначение |
|------|------------|
| *index.php* | Базовый шаблон, используется по умолчанию, если нет других более специфичных шаблонов. |
| *single.php* | Шаблон для отображения *одного поста* (записи блога). |
| *page.php* | Шаблон для отображения *одной страницы* (например, «О нас»). |

4. Зачем нужен файл functions.php в теме?

    Файл functions.php – это «плагин темы». Через него можно:

    - Подключать скрипты и стили (wp_enqueue_script, wp_enqueue_style).

    - Добавлять новые возможности темы (навигационные меню, виджеты, поддержка миниатюр, форматы постов).

    - Создавать пользовательские функции, фильтры и хуки.
